# PlyWP API Examples

Real-world code examples for using the PlyWP Client API and Application API in different programming languages based on actual API schemas.

## Client API Examples

### Authentication

All Client API requests require Bearer token authentication using your API key.

**Setup in cURL:**

```bash
export API_KEY="your_api_key_here"
export BASE_URL="http://localhost:5173/api/client"
```

**Setup in JavaScript:**

```javascript
const apiKey = process.env.REACT_APP_PLYWP_API_KEY;
const baseUrl = 'http://localhost:5173/api/client';

const headers = {
  'Authorization': `Bearer ${apiKey}`,
  'Content-Type': 'application/json'
};
```

**Setup in Python:**

```python
import os

API_KEY = os.environ.get('PLYWP_API_KEY', 'your_api_key_here')
BASE_URL = 'http://localhost:5173/api/client'

headers = {
    'Authorization': f'Bearer {API_KEY}',
    'Content-Type': 'application/json'
}
```

### Install and Activate Plugin

**cURL:**

```bash
curl -X POST "$BASE_URL/sites/{site_id}/plugins" \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "action": "install",
    "source": "jetpack",
    "activate": true
  }'
```

**JavaScript:**

```javascript
async function installPlugin(siteId, pluginSlug, autoActivate = true) {
  const response = await fetch(
    `${baseUrl}/sites/${siteId}/plugins`,
    {
      method: 'POST',
      headers,
      body: JSON.stringify({
        action: 'install',
        source: pluginSlug,
        activate: autoActivate
      })
    }
  );
  
  const result = await response.json();
  if (result.success) {
    console.log(`✓ Installed ${pluginSlug}`);
  } else {
    console.error(`✗ Failed to install ${pluginSlug}`);
  }
  
  return result;
}
```

## Automation Examples

### Automated Site Health Monitoring

**Python:**

```python
import requests
import time
from datetime import datetime

API_KEY = os.environ.get('PLYWP_API_KEY')
BASE_URL = 'http://localhost:5173/api/client'

def monitor_sites(site_ids, interval=300):
    """Monitor multiple sites every N seconds"""
    
    headers = {
        'Authorization': f'Bearer {API_KEY}',
        'Content-Type': 'application/json'
    }
    
    while True:
        for site_id in site_ids:
            try:
                response = requests.get(
                    f'{BASE_URL}/sites/{site_id}/health',
                    headers=headers,
                    timeout=10
                )
                
                health = response.json()
                timestamp = datetime.now().isoformat()
                
                if health.get('status') != 'healthy':
                    print(f"[{timestamp}] ⚠ {site_id}: {health.get('status')}")
                    
                    for warning in health.get('warnings', []):
                        print(f"  - {warning}")
                
            except Exception as e:
                print(f"[{timestamp}] ✗ Error checking {site_id}: {str(e)}")
        
        time.sleep(interval)

# Usage
monitor_sites(['site_123', 'site_456'], interval=300)  # Check every 5 minutes
```

## Error Handling

### JavaScript Error Handling

```javascript
async function apiCall(method, endpoint, body = null) {
  const headers = {
    'Authorization': `Bearer ${API_KEY}`,
    'Content-Type': 'application/json'
  };
  
  try {
    const options = { method, headers };
    if (body) options.body = JSON.stringify(body);
    
    const response = await fetch(`${baseUrl}${endpoint}`, options);
    const data = await response.json();
    
    if (!response.ok) {
      const error = data.error || {};
      throw new Error(
        `${response.status}: ${error.message || response.statusText}`
      );
    }
    
    return data;
  } catch (error) {
    console.error(`API Error (${endpoint}):`, error.message);
    throw error;
  }
}

// Usage with error handling
try {
  const sites = await apiCall('GET', '/sites');
  console.log('Sites:', sites);
} catch (error) {
  console.error('Failed to fetch sites:', error);
  // Implement fallback/retry logic here
}
```

### Python Error Handling

```python
def safe_api_call(method, endpoint, data=None):
    """Make API call with comprehensive error handling"""
    
    url = f'{BASE_URL}{endpoint}'
    
    try:
        if method == 'GET':
            response = requests.get(url, headers=headers, timeout=30)
        elif method == 'POST':
            response = requests.post(url, headers=headers, json=data, timeout=30)
        else:
            raise ValueError(f'Unsupported method: {method}')
        
        response.raise_for_status()
        return response.json()
    
    except requests.exceptions.Timeout:
        print(f'✗ Request timeout for {endpoint}')
        raise
    except requests.exceptions.ConnectionError:
        print(f'✗ Connection error for {endpoint}')
        raise
    except requests.exceptions.HTTPError as e:
        error_data = e.response.json() if e.response.text else {}
        print(f'✗ HTTP {e.response.status_code}: {error_data.get("error", {}).get("message")}')
        raise
    except ValueError as e:
        print(f'✗ Invalid response: {str(e)}')
        raise

# Usage
try:
    result = safe_api_call('GET', '/sites/site_123/health')
    print(f'Status: {result["status"]}')
except Exception as e:
    print(f'Operation failed: {e}')
```

## Best Practices

1. **Store credentials securely** - Use environment variables, never hardcode API keys
2. **Implement retry logic** - Handle rate limits with exponential backoff
3. **Use connection pooling** - Reuse HTTP connections for better performance
4. **Log all requests** - Keep audit trails for debugging and compliance
5. **Validate inputs** - Check data before sending to API
6. **Handle errors gracefully** - Provide meaningful error messages
7. **Cache responses** - Reduce API calls with appropriate caching
8. **Monitor rate limits** - Check X-RateLimit headers in responses
