# Architecture Overview

This guide explains the PlyWP architecture and how Panel and Plyorde work together to provide comprehensive WordPress management.

## High-Level Architecture

PlyWP consists of two main components:

1. **PlyWP Panel** - The web-based control interface for users
2. **Plyorde** - The system daemon that handles WordPress operations

These components communicate via a REST API to provide complete WordPress management functionality.

## PlyWP Panel

### Technology Stack

- **Framework**: Next.js 16 (React.js)
- **Language**: TypeScript
- **Styling**: Tailwind CSS
- **Database**: PostgreSQL
- **Authentication**: JWT-based tokens

### Core Responsibilities

1. **User Interface**
   - Dashboard showing all managed sites
   - Site management (create, update, delete)
   - Plugin and theme management UI
   - User management interface
   - Settings and configuration panels

2. **Authentication & Authorization**
   - User registration and login
   - Role-based access control (Admin, Editor, etc.)
   - API token generation and management
   - Session management

3. **Data Management**
   - Store user accounts and preferences
   - Store site configurations
   - Store API tokens and keys
   - Audit logging

4. **API Communication**
   - Communicate with Plyorde via REST API
   - Handle API errors and retries
   - Rate limiting management
   - Caching of WordPress data

## Plyorde Daemon

### Technology Stack

- **Language**: Go
- **Database**: SQLite or PostgreSQL
- **HTTP Framework**: Standard Go net/http or similar
- **Process Management**: Systemd (Linux)

### Core Responsibilities

1. **WordPress Discovery**
   - Scan configured directories for WordPress installations
   - Detect WordPress version and configuration
   - Verify database connectivity
   - Test WordPress health

2. **Plugin Management**
   - List installed plugins
   - Activate/deactivate plugins
   - Install/update/remove plugins
   - Check for available updates
   - Handle plugin conflicts

3. **User Management**
   - List WordPress users
   - Create/update/delete users
   - Manage user roles and capabilities
   - Reset passwords

4. **Database Operations**
   - Execute WordPress database queries
   - Optimize database
   - Monitor database size
   - Support database migrations

5. **File Operations**
   - Manage WordPress files and directories
   - Update WordPress core
   - Manage theme and plugin files
   - Handle file permissions

6. **Backup Management**
   - Create full and incremental backups
   - Store backup metadata
   - Restore from backups
   - Manage backup retention

7. **Monitoring & Health**
   - Monitor site uptime
   - Check SSL certificate validity
   - Track response times
   - Generate health reports

## Communication Flow

### Example: Creating a New Site

1. **User Action**: Administrator clicks "Add Site" in Panel
2. **Panel**: Displays site creation form
3. **User Input**: Fills in site details and submits
4. **Panel**: Validates input and calls Plyorde API:
   ```
   POST /api/v1/sites
   {
     "name": "My Site",
     "url": "https://mysite.com",
     "path": "/var/www/mysite.com",
     ...
   }
   ```
5. **Plyorde**: Receives request and:
   - Validates site path exists and has WordPress
   - Verifies database credentials
   - Checks for conflicts
   - Creates database record
   - Returns site details
6. **Panel**: Receives response and:
   - Displays success message
   - Refreshes site list
   - Redirects to site detail page
7. **User**: Sees new site in Panel dashboard

### Example: Updating a Plugin

1. **User Action**: Clicks "Update" for a plugin
2. **Panel**: Calls Plyorde API:
   ```
   POST /api/v1/sites/{site_id}/plugins/{plugin_id}/update
   ```
3. **Plyorde**: 
   - Downloads latest plugin version
   - Performs backup (if configured)
   - Extracts and installs plugin
   - Activates if previously active
   - Returns updated plugin info
4. **Panel**: Shows update completion to user

## Security Architecture

### Authentication

1. **User Login**
   - Panel authenticates user credentials
   - Generates JWT token
   - Returns token to client

2. **API Requests**
   - Client includes token in Authorization header
   - Plyorde validates token signature
   - Verifies token hasn't expired
   - Checks token permissions

3. **Token Refresh**
   - Short-lived access tokens (7 days)
   - Refresh tokens for obtaining new access tokens
   - API keys for long-lived access

### Authorization

1. **Role-Based Access Control**
   - Admin: Full access to all sites and features
   - Editor: Limited access to assigned sites
   - Viewer: Read-only access to assigned sites

2. **Resource-Level Permissions**
   - Users can only access sites they have access to
   - Actions (update, delete) require specific permissions

### Data Protection

1. **In Transit**
   - HTTPS for all communications
   - TLS certificates for encryption

2. **At Rest**
   - Database encryption (if configured)
   - API key hashing
   - Password hashing with bcrypt

### Production Setup

```
Internet
    │
    ├─ HTTPS (Port 443)
    │
    ▼
Reverse Proxy (Nginx/Apache)
    ├─ SSL Termination
    ├─ Load Balancing
    └─ Rate Limiting
    │
    ├─ Routes to Panel (Port 3000)
    │
    └─ Routes to Plyorde (Port 8081)
    │
    ├─ PostgreSQL Database
    │
    └─ Backup Storage
```

### High Availability

For production deployments:

1. **Panel Scaling**
   - Run multiple Panel instances
   - Use load balancer to distribute traffic
   - Share PostgreSQL database

2. **Plyorde Scaling**
   - Run Plyorde per server (or per group of sites)
   - Panel routes requests to appropriate Plyorde instance
   - Distributed backup storage
